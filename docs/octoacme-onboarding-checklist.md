# OctoAcme — New Team Member Onboarding Checklist

## Purpose
Help new team members ramp up quickly on OctoAcme project practices, tools, and team norms.

## Week 1 — Orientation

### Access & Tooling
- [ ] GitHub access granted and role confirmed (Developer, QA, etc.)
- [ ] Project board access confirmed (GitHub Projects or equivalent)
- [ ] Development environment set up (follow repo README)
- [ ] CI/CD pipeline walkthrough completed with DevOps / Platform Engineer
- [ ] Communication channels joined (Slack/Teams, email lists)

### Documentation Review
- [ ] Read [Project Management Overview](./octoacme-project-management-overview.md)
- [ ] Read [Roles & Personas](./octoacme-roles-and-personas.md) — identify your role(s)
- [ ] Read [Project Initiation Guide](./octoacme-project-initiation.md)
- [ ] Read [Project Planning](./octoacme-project-planning.md)
- [ ] Read [Execution & Tracking](./octoacme-execution-and-tracking.md)
- [ ] Read [Risk Management & Communication](./octoacme-risks-and-communication.md)
- [ ] Read [Release & Deployment Guide](./octoacme-release-and-deployment.md)
- [ ] Read [Retrospective & Continuous Improvement](./octoacme-retrospective-and-continuous-improvement.md)

### Introductions
- [ ] Met with Project Manager (PM) — understand current project status
- [ ] Met with Product Manager (PdM) — understand product goals and roadmap
- [ ] Met with Tech Lead — understand architecture and technical conventions
- [ ] Introduced in team standup

---

## Week 2 — Active Participation

### Process Integration
- [ ] Attended at least one standup
- [ ] Attended sprint planning or backlog refinement
- [ ] Reviewed the current Risk Register with PM
- [ ] Reviewed current project board and sprint backlog
- [ ] Submitted first PR (even a small doc fix counts) and had it reviewed

### Role-Specific Steps

#### Developers
- [ ] Reviewed branching and PR conventions in the repo
- [ ] Completed first code review on an existing PR
- [ ] Confirmed local test suite runs successfully

#### QA / Test Engineers
- [ ] Reviewed existing test plans and test suites
- [ ] Confirmed test environment access and smoke test execution
- [ ] Met with DevOps to understand environment provisioning

#### UX / Product Designers
- [ ] Gained access to design tool (Figma or equivalent)
- [ ] Reviewed existing design system or component library
- [ ] Met with Product Manager to review current UX research artifacts

#### DevOps / Platform Engineers
- [ ] Completed infrastructure and environment runbook review
- [ ] Confirmed access to deployment pipelines and monitoring dashboards
- [ ] Reviewed incident response and on-call procedures

---

## Week 3–4 — Full Contribution

- [ ] Independently picking up and completing backlog items
- [ ] Actively contributing in retrospectives and demos
- [ ] Questions answered and blockers resolved — escalate to PM if not
- [ ] 30-day check-in with PM or Tech Lead to surface any gaps or concerns
